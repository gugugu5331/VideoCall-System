# 贴图/虚拟形象功能现状

当前 Web 客户端未内置贴图、人脸挂件或虚拟形象功能，后端亦无相关接口/存储。若需要扩展，请遵循下述约束以避免影响现有链路。

- **渲染方式**：在浏览器本地使用 Canvas/WebGL 或 WebRTC Insertable Streams 叠加贴图，尽量保持 SFU 仅转发码流。
- **数据来源**：本地静态资源或对象存储（MinIO）；上传后返回公开 URL 供前端引用，不必修改后端 schema。
- **定位与同步**：可结合 FaceMesh/FaceDetector 等在端侧计算关键点，仅广播渲染参数（位置/缩放/旋转）至房间，其余参与者本地渲染。
- **权限与隐私**：避免将人脸特征上传后端；如需落库，请新增独立表/桶并确保鉴权。
- **UI 处理**：在功能落地前隐藏或禁用入口，保持与实际能力一致。
