# 贴图/虚拟形象功能说明（当前状态）

当前仓库的 Web 客户端未实现贴图/人脸挂件功能，代码中亦无相关资源或接口。若后续需要支持，可按以下方向扩展，避免与现有架构冲突：

- **渲染方式**：在浏览器端使用 `Canvas`/`WebGL` 或 `WebRTC Insertable Streams` 叠加贴图，保持 SFU 不改码流。
- **数据来源**：本地静态资源或对象存储（MinIO）；上传后返回可公开访问的 URL。
- **锚点定位**：结合 `FaceMesh/MediaPipe` 或浏览器内置 `FaceDetector`，在本地计算关键点，不将人脸特征上传到后端。
- **同步策略**：仅在客户端渲染，或通过信令将参数（位置/缩放/旋转）广播给其他参会者；无需改动媒体服务。
- **权限与存储**：由客户端自行管理，当前后端未提供贴图元数据表。

待实现前，UI 可隐藏或禁用相关入口，以保持与实际能力一致。
